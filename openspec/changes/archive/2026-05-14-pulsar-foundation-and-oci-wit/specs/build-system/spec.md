## ADDED Requirements

### Requirement: Root Cargo Workspace
The repository SHALL define a root Cargo workspace that includes every FaaS crate and `pulsar-cli`, centralizing common dependency versions.

#### Scenario: Workspace metadata is requested
- **GIVEN** the repository root contains `Cargo.toml`
- **WHEN** `cargo metadata` runs from the root
- **THEN** all FaaS crates and `pulsar-cli` are discovered as workspace members

### Requirement: Pinned Rust Toolchain
The repository SHALL pin Rust `1.85.0` with `rustfmt`, `clippy`, and the `wasm32-wasip1` target.

#### Scenario: Developer builds locally
- **GIVEN** `rust-toolchain.toml` is present at the repository root
- **WHEN** a developer runs Cargo commands
- **THEN** Rustup selects the pinned compiler and installs the declared target as needed

### Requirement: Stable Rust Editions
All Rust package manifests under `faas/*` and `pulsar-cli` SHALL use edition `2021`.

#### Scenario: Cargo validates manifests
- **GIVEN** workspace package manifests exist
- **WHEN** Cargo parses them
- **THEN** no package declares edition `2024`

### Requirement: Workspace Dependency Inheritance
Leaf package manifests SHALL use `{ workspace = true }` for shared crates so versions are resolved once at the root.

#### Scenario: Shared dependency changes
- **GIVEN** a shared dependency version is updated in the root workspace
- **WHEN** workspace packages are built
- **THEN** each leaf package inherits the updated version

### Requirement: OCI WIT Dependency Metadata
Each FaaS package SHALL declare component metadata pointing Tachyon WIT dependencies at `oci://ghcr.io/astorise/tachyon-mesh-wit:0.9.0-rc.1`.

#### Scenario: Component metadata is inspected
- **GIVEN** a FaaS package manifest is opened
- **WHEN** its component metadata is read
- **THEN** the Tachyon WIT dependency source is the pinned OCI artifact

### Requirement: Unified Lockfile
The repository SHALL commit a root `Cargo.lock` generated by `cargo fetch` from the workspace root.

#### Scenario: CI resolves dependencies
- **GIVEN** the root lockfile exists
- **WHEN** CI builds the workspace
- **THEN** dependency versions are resolved deterministically
